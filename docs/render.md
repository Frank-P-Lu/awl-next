# awl docs — Render: adaptive column, theme caps, overlay chrome, picker rows

> Read before touching `render/`, `theme/model.rs` (RenderCaps), `overlay/`, `settings.rs` palette rows, or picker/row layout. Moved verbatim out of CLAUDE.md 2026-07-22 (queue item 17); earlier round history: `git log -p CLAUDE.md`.

## Adaptive-column placement (`render/geometry.rs`)

- On a small screen the symmetric centered column cramped the margin outline while the right margin sat empty. `TextPipeline::column_left` is the one owner of an adaptive policy (no config knob): shifts right under pressure to grant the outline a rail (taking the empty right-margin space), back to symmetric when there's room. Every downstream reader (caret/selection/washes, hit-test, drag handle, gutter) composes it for free.
- Pure policy `adaptive_column_left`, one formula `desired_left.min(max_left).max(symmetric_left)`, three regimes (wide = byte-identical passthrough to the old column; narrow = shift, width never touched; narrowest = re-centers → outline auto-hides at its existing floor). Hide threshold + shift threshold read the same `left` (can't drift). Transition is instant (glide banked). **Live-only:** the feel of a real small-screen resize.

## Theme capabilities as data (`theme/model.rs::RenderCaps`)

- Render call sites never branch on world identity — they read a `RenderCaps` field: `selection_style`, `caret_block_style`, `backdrop`, `elevation`, `decorative_wash`, `image_reveal`, `highlight_texture`, plus the personality fields `card_anchor`, `chrome_face`, `motion`, `list_style`, `facet_style` (and `TitleStyle::Placard{corner,scale,ink}`). Born as the Wagtail refactor ("default = every other world"); that framing is dead — 16 worlds now, and many set fields away from default (Firetail: placard BL 4.5/Bold + `chrome_face` Named("Archivo Black") + Bordered elevation; Galah/Magpie/Mangrove/Firetail: `list_style` Bars + placards; five worlds TopLeft `card_anchor` (Cassowary + Mangrove now `TopRight` — the item-45 fable picks) while default flipped TopCenter; lava worlds `motion` calm). **No theme may need its own code path** — new personality = new caps field + data.
- **ITEM 70's PRINTED-CARD caps** (`card_texture: CardTexture::Flat|HalftoneDots{angle_deg,cell_px,density}`, `card_shape: CardShape::Rectangular|Chamfered{cut_px}`) — Quokka alone assigns the non-defaults. Read by the ONE `ListBacking::Card` consumer (`render::chrome::TextPipeline::card_shape_texture`, called from `prepare_panel_card_elevation` + `overlay_draw_card`'s spell-popup arm): a plain `sd_round_rect` SDF (`shaders/selection.wgsl`) gets replaced by `sd_card_rect`'s CSG intersection with a diagonal half-plane when `chamfer > 0`, shared by the fill/border/shadow trio so the eight-edge boundary agrees across all three; the dot lattice composites only over the FILL, sampled at the ABSOLUTE canvas pixel (not instance-local) so a Split Pane's two surfaces share one continuous phase across the open gap, with a pure horizontal rolloff (`halftone_rolloff`) keeping the left content column clear. The dot's own ink is DERIVED (`theme::card_texture_ink`, a `muted`→`base_300` blend) — never a raw/amber literal in the cap itself. `render::chrome::narrowed_chamfer_px` caps the chamfer at 40% of a card's own smaller dimension so a genuinely tiny surface (the spell popup) never lets the cut steal text room, while an ordinary short query bar stays untouched. Dev-only gallery knob `AWL_CARD_CAPS_FORCE=flat|halftone` (mirrors `AWL_CJK_FORCE`).
- `is_one_bit()` still exists (pins Wagtail's identity for the monochrome law tests) but the renderer no longer reads it. **Grep-law `theme_caps_law`:** fails if `.is_one_bit(` or a quoted world name appears in real code under `src/render/` — structurally bans a future per-theme special case.
- **Lava frost is an ORGANIC glyph-seeded FIELD (item 32), not rectangular pills.** The lava worlds (Mangrove + Firetail) keep both margins alive and frost the drawn margin ink (outline + gutter). Every visible margin glyph SEEDS a small close halo `[x0, x1, yc, r]`; the shader SUMS the halos into one continuous scalar field and thresholds it (`lava::frost_coverage`), so neighbours MERGE in any direction — nearby words/rows become one organic island, a group-gapped row separates, with NO per-row rule and NO zoom breakpoint (topology falls out of the continuous field). The ROW's radius ceiling is derived from the ZOOMED LABEL row height + the `feather_px` skirt (`render::frost_seed_radius`), and the skirt being a fixed logical px while the row scales with zoom is what lets islands join at small zoom and split at large zoom, naturally. Both worlds share the SAME shape/blur/dim/feather recipe (`crate::lava` consts); only the palette-derived colour differs (the world's own `Background::Lava` ground/blobs, via `render_caps.frost`). Owners: `TextPipeline::outline_frost_seeds` + `gutter_frost_seeds` → the shared `render::push_text_seeds` (per-glyph or the word-run degradation arm, `lava::FROST_SEED_PER_GLYPH`); the seed field is a PROTO-CACHE (`frost_seed_key` / `frost_seeds`, rebuilt only on a viewport/zoom/column/margin-text miss — witnessed by `--bench-frost`). All non-lava worlds stay byte-identical (empty seeds → inert shader path).
- **EACH SEED's own radius is PUNCTUATION-AWARE and BOUNDED, not a blanket row value (item 61).** The row's `frost_seed_radius` is a CEILING, not every run's literal radius: `render::frost_run_radius(r_row, run_ink_w, skirt)` takes the tightest of three — `r_row` itself, the run's OWN measured ink half-width (× `lava::FROST_RUN_INK_RADIUS_FRAC`) plus the skirt, and a BOUNDED END-PAD ceiling (`skirt × lava::FROST_END_RADIUS_SKIRTS`, independent of row height). An isotropic seed's `(1-(d/r)^2)^2` falloff is symmetric, so a near-zero-ink run at the FULL row-height radius reads as a bare circle hung off the glyph (an isolated `&`, a one-letter heading) instead of a close hug — the ink-derived bound ties that run's own radius to ITS OWN advance geometry instead. The end-pad ceiling separately caps how far a LONG single-run label's halo can reach past its own final glyph (a hyphenated compound like "Button-free" has no internal whitespace to break it into shorter runs) — a bound that does NOT grow with a taller row (a deep heading-ladder rung, a big zoom), only with the skirt. An ordinary multi-glyph run's ink half-width and the end-pad ceiling both already sit at or above `r_row` in practice, so `min()` is a no-op there — row/nearby-run merging for ordinary text is unchanged. No glyph or label is identity-matched; the bound is purely geometric. Laws: `render/tests/frost.rs`'s `isolated_punctuation_run_radius_is_bounded_below_a_normal_runs`, `long_run_end_pad_is_bounded_independent_of_row_height`, `nearby_ordinary_runs_still_merge_after_the_bounded_radius_round`.

## Overlay personality & chrome composition (render/chrome/ + theme/model.rs)

- A fortnight of rounds, one shape: overlay/chrome variety is data in `RenderCaps`, never a per-world code path. `ListStyle` (Pane default | Bars = per-row plates), `FacetStyle` (Text default | Band | Chips), `CardAnchor` (incl. TopRight + `mirrors_growth` for Bars), `TitleStyle::Placard{PlacardCorner, PlacardInk}`.
- **`PaneSplit` (Split default | Unified) — the SPLIT-PANE composition (item 50).** A Pane world's takeover card draws as TWO surfaces: the title/query INPUT on the upper surface, a visible strip of the world's own background between, then ONE lower surface owning facets/section-headers + candidate rows + footer. DATA-gated through the ONE owner `render::effective_pane_split` (probe `AWL_PANE_SPLIT_FORCE=unified|split`); the fill rides `overlay_pane_fills` → `overlay_split_bounds` (the gap is carved from the query beat's own negative space, so NO text moves and no glyph falls in the gap — a pure fill change). Cassowary is the ONE `Unified` exception (dormant under its Bars list). Inert on a Bars world (bare plates) and the spell popup. Two bordered surfaces on a 1-bit world (Wagtail) read by their rims. Laws: `render/tests/split_pane.rs`.
- **Held back, not dead:** Chips is rebuilt-for-real but ships inert pending the user's variant pick; poster facets stay Text. Probe forces for galleries: `AWL_FACET_STYLE_FORCE`, `AWL_OVERLAY_ANCHOR_FORCE` (env, CLI-invisible).
- **`PlacardCorner::Auto` derives complementary to the card anchor via one owner `render::derived_placard_corner`;** `overlay_shape_placard` shrinks-to-fit so placards never clip — the old "every placard BL" pin is retired for an end-to-end no-clip outcome law.
- **One-owner geometry seams (route through these, never re-derive):** `overlay_card_x`, `overlay_row_top`/`_of`/`_index` (+ `header_gap`), `overlay_split_bounds`/`overlay_pane_fills` (the split surfaces), `push_overlay_hint_spans`, `overlay_footer_reclaim`.
- **THE INTERIOR-RAIL PLACEMENT LAW (item 67) — one global resolver, `render::chrome::overlay::overlay_rail_inset(ww)`.** Supersedes the old flush ~28px `TopLeft`/`TopRight` edge-hug: the two side rails are now SYMMETRIC INTERIOR rails whose card CENTERS sit near the viewport's one-third / two-thirds marks when the card holds a comfortable width — generous outer breathing room, still unmistakably left/right. `TopCenter` is untouched (`free * 0.5`, exactly the viewport midpoint, always). The three theme-owned anchor CHOICES and every world's own assignment are unchanged; only the shared placement arithmetic moved. The inset is a PURE function of the window width alone — never the anchor, never the caller's actual `desired_w` — so `overlay_card_box_policy`'s `TopLeft` and `TopRight` arms read the exact same number and the left/right MIRROR law holds by construction, independent of content width (item 51's right-anchor content-hug still shares ONE right edge across a plain and a secondary-bearing card). The SAME `full -> anchored_max -> floor -> free` clamp chain the old fixed constant fed keeps the narrow-window collapse toward `CARD_EDGE_INSET_FLOOR` (then the re-centering fill regime) perfectly CONTINUOUS — no breakpoint jump, no per-anchor branch, no fourth user-facing anchor. `CardAnchor::Inset { x_frac }` keeps its exact `0.0 == TopLeft`, `0.5 == TopCenter`, `1.0 == TopRight` contract untouched (the same clamp expressions, just reading the new inset).
- **The card ANCHOR is frozen at summon but RE-STAMPED on a deliberate theme crossing (item 45 → item 52).** The render path reads the anchor ONLY through the frozen `resolve_overlay_anchor(self.overlay_align)` — never the live world (`render::tests::overlay_align_law`'s alignment-is-data grep-law bans `effective_card_anchor(`/`render_caps.card_anchor` anywhere under `src/render/`). Item 52 makes the theme picker CROSS its anchor like every other theme-owned property: `OverlayState::reanchor` (called by `actions::preview_move` on a keyboard nav / wheel move, and the wheel path in `app/input/mouse.rs`) re-stamps `align` to the highlighted world's anchor AFTER the preview makes it active, so the card SNAPS into the destination rail. Passive `overlay_hover` keeps the BARE `preview_overlay` (no reanchor — no spatial chase). The re-stamp lives in the overlay/actions layer, NOT a render consumer, so the grep-law still holds. Laws: `render::tests::reanchor_crossing_law`.
- **Right-anchored cards shrink to content (item 51):** a `TopRight` (`mirrors_growth`) takeover card no longer seats a wide `CARD_MAX_W` card against the right edge (the 300–400px dead middle between the left-aligned labels and the remote edge). `overlay_desired_w` (the ONE desired-width owner both geometry paths read) returns the MEASURED content width — `measure_overlay_content_w`, cached in `overlay_content_w` at sync via the SAME real shaper the draw runs (widest visible primary + optional secondary column + query/strip/footer + `2·hpad`), clamped to `[CARD_CONTENT_MIN_W, wide cap]` — so the whole group hugs the right window edge as one block, text still LEFT-aligned inside, the secondary right-aligned to the card's own text edge (a content-bounded scanning column, never the remote edge). Left/center keep `overlay_content_w == 0.0` → the fixed wide cap, byte-identical. Laws: `render::tests::overlay_right_hug_law`.
- The theme picker retired its runtime lens strip (user decision 2026-07-15, recorded in src/facets.rs); the axes are a build-time ruler.

## Settings in the palette + overlay titles (`overlay/` + `settings.rs`)

- The Cmd-P palette's rows are catalog commands **∪** `settings::SETTINGS` (a settings row like "Keymap" is fuzzy-findable straight from the palette). Still one `OverlayKind::Command`; the union is data (`attach_settings_rows`, an `is_setting` flag). A settings row shows its current value in the secondary column; marker prefix `§ ` (measured bundled in `AwlMarks.ttf`; the gear ⚙ is not bundled, so it never competed). Dispatch parity via one owner `dispatch_settings_row` (`close_on_toggle` = the only difference: palette closes, Settings menu stays).
- **Every `OverlayKind` names itself** (`OverlayKind::title`, no-wildcard) — drawn as a muted prefix on the picker's input line (Rename/InsertLink opt out via `draws_title_prefix`, their own prompt orients). Sidecar `overlay.title`.

## Picker rows (`render/rowlayout`)

Picker rows go through `render/rowlayout` — never place row text directly. A primary cell (never dropped, elided last-resort) + optional secondary right column (first to yield). `rowlayout::plan` → `fits` → `fit_primary` (the only elision door). The law test enumerates `OverlayKind` with a no-wildcard match. The bottom-left page-mode gutter rides the same owner (`gutter_plan`) — stacked, so neither line yields from width pressure (the filename never wraps; the fix for "DESIGN.md → DESIG/N.md and the project vanishes").
